01 — The problem

The clipboard holds one thing. You copy hundreds.

Every copy overwrites the last. The command you ran ten minutes ago, the path you keep retyping, the snippet you needed twice — gone the moment you copy something else. So you switch windows, scroll back, and copy it again.

overwritten
The thing you just copied replaces the thing you needed.
repeated
Copy, paste, lose it, find it, copy again. All day.

04 — Pinning & sessions

From passing history to a kit you keep.

Pin the snippets you reuse so they outlive the history limit. Reorder them, label them, hide sensitive ones. Group related pins into named sessions — a deploy runbook, a set of prompts, the commands for one project.

Pin & reorder Custom labels Hide sensitive content Named sessions Always-on Favorites
05 — Search & recall

Filter as you type. Find it in two keys.

Search runs across pinned items and history at once. Jump straight to the search box with a shortcut, narrow the list instantly, and act without leaving the keyboard. Right-click any item for context actions — copy, label, or pin it directly into a session.

→Instant filtering across pinned and history.
→Jump to search with a single shortcut.
→Context menu actions, including pin-to-session.
06 — Bonus: app switcher

A second hotkey, for switching apps.

ClipX ships a separate global hotkey that opens a list of running apps. Type to filter by name, then activate with Enter, a number key, or a click. Same keyboard-first habit, applied to your whole desktop.

Open list of apps ⌃ Control ⌥ Option Esc
Activate with Enter, a number key, or a click.
07 — Local-first

Your clipboard never leaves the machine.

History is stored locally in a SQLite database on your disk. There's no account, no cloud sync, and no clipboard content sent anywhere. ClipX is a quiet local tool — what you copy stays with you.

✓Stored on-device in SQLite
✓No cloud dependency, no account
✓Nothing logged or sent off the machine
History limit is yours — keep up to 50 recent entries.
